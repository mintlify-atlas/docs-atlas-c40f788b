---
title: Credential Broker
description: Just-in-time credential injection with rotation-aware caching and zero-persistence guarantees
---

Faramesh's **Credential Broker** ensures agents never hold long-lived credentials. For each permitted tool call, the broker fetches the minimal credential needed, injects it for the duration of the call, then discards it.

<Warning>
Credential values are **NEVER** logged, written to DPR records, or included in telemetry.
</Warning>

---

### Key Properties

- **Zero persistence** вҖ” Credentials never touch disk, logs, or DPR
- **Minimal scope** вҖ” Each credential is scoped to one tool operation
- **Automatic revocation** вҖ” Credentials are revoked immediately after use
- **Backend agnostic** вҖ” Pluggable backends for Vault, AWS, GCP, Azure

---

## Broker Interface

All credential backends implement the `Broker` interface:

```go
type Broker interface {
    // Fetch retrieves a scoped credential for one tool call
    Fetch(ctx context.Context, req FetchRequest) (*Credential, error)
    
    // Revoke revokes the credential after use
    Revoke(ctx context.Context, cred *Credential) error
    
    // Name returns the backend identifier
    Name() string
}
```

### Credential Type

```go
type Credential struct {
    Value     string    // NEVER LOGGED
    Source    string    // Backend name ("vault", "aws")
    Scope     string    // Permission scope granted
    ExpiresAt time.Time // Credential expiry
    Revocable bool      // Can it be revoked?
}
```

## Supported Backends

### HashiCorp Vault

Vault provides dynamic secrets, PKI certificates, and cloud credentials.

**Configuration:**
```yaml
credential_broker:
  backend: vault
  vault:
    addr: https://vault.company.com
    token: ${VAULT_TOKEN}
    
routes:
  stripe/*: vault          # Stripe tools use Vault
  aws/*: vault             # AWS tools get STS tokens from Vault
  database/*: vault        # Database gets dynamic creds
```

**Supported Vault Paths:**
- `aws/sts/{role}` вҖ” AWS STS tokens
- `database/creds/{role}` вҖ” Dynamic database credentials
- `pki/issue/{role}` вҖ” X.509 certificates
- `kv/data/{path}` вҖ” Static secrets (discouraged)

## Workload Identity

Faramesh supports cloud-native workload identity to authenticate with secret backends **without long-lived credentials**.

### SPIFFE/SPIRE

SPIFFE provides mTLS-based identity using Verifiable Identity Documents (SVIDs).

**Configuration:**
```yaml
workload_identity:
  enabled: true
  provider: spiffe
  spiffe:
    workload_api_socket: unix:///run/spire/sockets/agent.sock
    trust_domain: example.org
```

**How it works:**

1. Faramesh daemon connects to SPIRE Workload API
2. SPIRE issues an X.509 SVID (auto-rotated)
3. SVID is used for mTLS auth to Vault, databases, etc.
4. Trust bundles auto-rotate via SPIRE

**Policy integration:**
```yaml
rules:
  - id: require-verified-identity
    match:
      tool: db/write
      when: 'principal.method != "spiffe"'
    effect: deny
    reason: database writes require SPIFFE identity
```

## Credential Router

The `Router` maps tools to credential backends.

### Route Configuration

```yaml
routes:
  # Exact match
  stripe/refund: vault
  
  # Glob patterns
  stripe/*: vault
  aws/*: vault
  db/*: vault
  
  # Fallback
  "*": env
```

### Fallback to Environment

If no route matches, the `EnvBroker` is used (credentials from environment variables).

<Warning>
Environment-based credentials are **less secure** than brokered credentials because they persist for the deployment lifetime.
</Warning>

## SecretBuffer with mlock

In production, credential values are stored in **memory-locked buffers** to prevent swapping to disk.

```go
// In production: use mlock to prevent paging
type SecretBuffer struct {
    data []byte
    locked bool
}

func (sb *SecretBuffer) Lock() error {
    return syscall.Mlock(sb.data)
}

func (sb *SecretBuffer) Zero() {
    for i := range sb.data {
        sb.data[i] = 0
    }
}
```

## Rotation-Aware Caching

Faramesh caches credentials **within their TTL** but monitors for rotation events.

### Cache Behavior

1. **Fetch credential** from backend (e.g., Vault)
2. **Cache** until `ExpiresAt - 30s`
3. **Monitor** for rotation signals (Vault lease renewal)
4. **Evict** on rotation or expiry

```yaml
credential_cache:
  enabled: true
  max_ttl: 5m
  refresh_threshold: 30s  # Refresh 30s before expiry
```

## DPR Metadata

Only **metadata** about credential brokering is recorded in DPR:

```json
{
  "credential_brokered": true,
  "credential_source": "vault",
  "credential_scope": "stripe:charges:write",
  "credential_ttl": "5m"
}
```

The credential **value** is never included.

## Lifecycle Example

```go
// 1. Policy permits the call
decision := engine.Evaluate(req)
if decision.Effect != "PERMIT" {
    return decision
}

// 2. Router resolves the backend
router := credential.NewRouter(backends, fallback)
broker := router.Resolve(req.ToolID)

// 3. Fetch credential
handle, err := router.BrokerCall(ctx, credential.FetchRequest{
    ToolID:    req.ToolID,
    Operation: "create",
    Scope:     "stripe:charges:write",
    AgentID:   req.AgentID,
    TTL:       5 * time.Minute,
})
if err != nil {
    return &Decision{Effect: "DENY", Reason: err.Error()}
}
defer handle.Release(ctx)  // Revoke and zero

// 4. Inject into tool environment
os.Setenv("STRIPE_API_KEY", handle.Credential.Value)

// 5. Execute tool
result := executeTool(req)

// 6. Credential is automatically revoked and zeroed by defer
return result
```

## Best Practices

<Info>
**Use workload identity whenever possible** вҖ” Eliminates bootstrap credentials.
</Info>

1. **Never use static credentials** вҖ” Use dynamic secrets from Vault or cloud providers
2. **Scope credentials minimally** вҖ” One tool, one operation, one credential
3. **Set short TTLs** вҖ” 5 minutes is usually sufficient
4. **Monitor for rotation** вҖ” Evict cached credentials on rotation events
5. **Use SPIFFE in Kubernetes** вҖ” mTLS-based identity without shared secrets

<Warning>
**Avoid the `env` broker in production** вҖ” It's a fallback for development only.
</Warning>
